Python f-Strings

The Modern Way to Format Strings in Python

f-strings (formatted string literals) were introduced in Python 3.6 and have become the go-to method for string formatting. They're fast, readable, and incredibly flexible.

However, when partnered with user input, they can be a liability...

1. Basic Syntax

Prefix your string with f and insert variables using curly braces:

Unlike when concatenating variables, you do not need to cast all your variables to strings with str().

name = "Alice"
age = 30
print(f"My name is {name} and I'm {age} years old.")

Output:

My name is Alice and I'm 30 years old.

2. Expressions Inside f-Strings

You can use expressions directly inside curly braces:

print(f"5 + 3 = {5 + 3}")

Output:

5 + 3 = 8

3. Function Calls

Functions can be called inside f-strings:

def greet(name):
    return f"Hello, {name}!"

print(f"{greet('Bob')}")

Output:

Hello, Bob!

4. Formatting Numbers

Control decimal places by adding `:.2f` after your variable, changing the number to fit your desired number of decimal points:

pi = 3.1415926535
print(f"Pi to 2 decimal places: {pi:.2f}")

Output:

Pi to 2 decimal places: 3.14

5. Padding and Alignment

Align text using <, ^, and > with width:

print(f"{'Left':<10}{'Center':^10}{'Right':>10}")

Output:

Left      Center      Right

Instead of padding with spaces, you can also pad with any character by putting it before the arrow symbol:

print(f"{'Left':-<30}")
print(f"{'Center':_^30}")
print(f"{'Right':#>30}")

Output:

Left--------------------------
____________Center____________
#########################Right

6. Debugging with =

Python 3.8+ allows = for quick debugging:

value = 42
print(f"{value=}")

Output:

value=42

7. Multiline f-Strings

Use triple quotes for long or multiline strings:

name = "Eve"
age = 22
print(f"""Name: {name}
Age: {age}""")

Output:

Name: Eve
Age: 22

⚠️ f-String Dangers

f-strings are powerful, but dangerous if used with untrusted input.


SQL Injection

Don't build SQL with f-strings. It can allow attackers to run harmful SQL.

❌ Unsafe:

cursor.execute(f"SELECT * FROM users WHERE username = '{username}'")

This could delete your entire database.

✅ Safe:

cursor.execute("SELECT * FROM users WHERE username = ?", (username,))

Code Injection

Never run f-strings with eval() or exec() on user input.

❌ Unsafe:

eval(f"print({user_input})")

The user can easily get out of the print by entering the correct input, allowing them to run actual harmful code instead.

✅ Safe: Don’t use eval() unless absolutely necessary. Avoid running raw input as code, make sure to sanitise it first.

Conclusion

f-Strings are one of the best additions to modern Python. They're readable, flexible, and powerful for string formatting, debugging, and data presentation. Start using them to make your code cleaner and quicker.
